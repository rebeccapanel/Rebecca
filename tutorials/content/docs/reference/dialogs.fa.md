## اطلاعات پایه و وضعیت {#dialog-dialog-basics}

بخش بالای فرم برای اسم، وضعیت و زمان/حجم است.

### Username

نام کاربری بین ۳ تا ۳۲ کاراکتر؛ بهتر است ساده و بدون فاصله باشد.

- برای تست از پسوند trial/guest استفاده کن تا در لیست گم نشود.

### Status

active برای استفاده عادی، on hold برای شروع بعد از اولین اتصال، limited/expired/disabled برای قطع یا محدودسازی.

- on hold تاریخ و حجم را تا اولین اتصال نگه می‌دارد.

### Expire

تاریخ پایان دسترسی؛ اگر خالی بگذاری تاریخ ندارد.

- گزینه «+30 روز» و ... برای سریع‌تر کردن تمدید بالای جدول هست.

### Data limit (GB)

سقف حجم بر حسب گیگابایت؛ اگر خالی بگذاری نامحدود است.

- برای سرویس تست 10 تا 50 گیگ کافی است.

### IP limit

تعداد IP مجاز همزمان؛ روی ۱ یا ۲ بگذار تا اشتراک پخش نشود.

## سرویس و ورودی‌ها {#dialog-dialog-service}

اینجا تعیین می‌کنی لینک از کدام inbound و پروتکل ساخته شود.

### Service

اگر از پلن‌های از پیش ساخته استفاده می‌کنی، یک سرویس انتخاب کن تا تنظیمات از آن خوانده شود.

### Inbounds / پروتکل‌ها

برای هر پروتکل (vless/vmess/trojan/ss) یک ورودی انتخاب کن تا لینک کاربر ساخته شود.

- اگر لیست خالی بود، اول در تنظیمات ورودی (Hosts/Nodes) یک inbound اضافه کن.
- اگر از setservice-* استفاده می‌کنی، باید فقط همان تگ انتخاب شود تا سرویس خودکار ست شود.
- وقتی سرویس را دستی انتخاب کنی، انتخاب inboundها نادیده گرفته می‌شود.

### Flow / TLS

اگر لازم است flow خاص (مثل xtls-rprx-vision) یا رمز سفارشی وارد کن؛ در غیر این صورت خالی بگذار.

## کلید دسترسی و لینک‌ها {#dialog-dialog-access}

گزینه‌های مربوط به لینک و credential key.

### Manual credential key

اگر می‌خواهی کلید ۳۲ رقمی سفارشی بدهی، تیک Manual را بزن و مقدار را وارد کن.

### Subscription link / QR

بعد از ذخیره، از جدول کاربران روی آیکن لینک یا QR بزن تا لینک تازه بسازی.

## تمدید خودکار (Auto Renew) {#dialog-dialog-auto-renew}

برای تمدید زمان یا حجم بدون دخالت دستی.

### Data limit

حجم جدیدی که بعد از رسیدن به سقف اعمال می‌شود.

- گزینه add remaining ترافیک باقی‌مانده را به دوره بعد منتقل می‌کند.

### Time limit (days)

چند روز به تاریخ اضافه شود؛ صفر یعنی بدون محدودیت زمانی.

### Fire on either

اگر فعال باشد، با تمام شدن زمان یا حجم هرکدام زودتر برسد تمدید می‌کند.

## سایر اطلاعات و پشتیبانی {#dialog-dialog-other}

فیلدهای تماس و یادداشت برای مدیریت راحت‌تر.

### Note

توضیح کوتاه درباره نوع کاربر یا توافق؛ بعدا هنگام جستجو کمک می‌کند.

### Telegram ID / Contact number

اطلاعات ارتباطی برای پیگیری سریع.
