{{< details title="When should I use on hold?" closed="true" >}}

<span id="faq-when-onhold"></span>

Whenever you don’t want the account to start until the customer actually connects.

- Trial accounts that might never connect.
- Customers who need to pay first, then activate.
- Cases where you want to send the link early but start counting later.

{{< /details >}}

{{< details title="What’s the fastest way to renew?" closed="true" >}}

<span id="faq-fast-renew"></span>

Open the user dialog, change the expiry or quota, and save.

- Turn on Auto Renew in the same dialog for hands-off renewals.
- Use reset usage if you want to zero out their counters.

{{< /details >}}

{{< details title="What’s the difference between active, on hold, and disabled?" closed="true" >}}

<span id="faq-status-differences"></span>

Active = full access; on hold = waiting for the first connection; disabled = paused until you change it back.

{{< /details >}}

{{< details title="User says they can’t connect—where do I start?" closed="true" >}}

<span id="faq-troubleshoot"></span>

Check three things: status is active, expiry/quota aren’t blocking, and the link is current.

- Grab a fresh QR or subscription link from the table.
- If status is limited or expired, extend it as needed.
- Lower the IP limit and reset usage if you suspect sharing.

{{< /details >}}

{{< details title="What is a subscription and where do I create it?" closed="true" >}}

<span id="faq-subscription-basics"></span>

A subscription is the user account + link. Go to the Users page and click Create user to make one.

- Data limit = quota cap (GB).
- Time limit = expiry date; access stops after it.
- Service = prebuilt plan with inbound/quota/expiry defaults.

{{< /details >}}

{{< details title="Common questions about subscription settings" closed="true" >}}

<span id="faq-subscription-advanced"></span>

Definitions you’ll see when creating/editing:

- Auto delete: remove the user after time/quota ends.
- Auto renew: renews traffic/time automatically.
- Add remaining: carry leftover traffic into the next cycle.
- Revoke access: invalidate old links (revoke subscription).
- Reset usage: zero out usage counters.
- Enable/disable: temporarily allow or block access.
- How does it connect? Via the subscription link/QR in your V2Ray/Reality/SS client.

{{< /details >}}
