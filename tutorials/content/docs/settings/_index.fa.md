صفحه تنظیمات رفتارهای مشترک پنل و یکپارچه‌سازی‌ها را کنترل می‌کند. تب‌ها و عملیات قابل مشاهده به مجوز ادمین و نوع نصب بستگی دارند.

<p class="rb-panel-actions"><a class="rb-panel-button" data-primary="true" href="#" data-panel-route="/settings#panel">باز کردن تنظیمات</a></p>

## پیش از هر تغییر

- تنظیمات این صفحه مشترک‌اند؛ یک تغییر می‌تواند روی ادمین‌های دیگر، لینک‌های تولیدشده، تاریخچه مصرف یا عملیات نودها اثر بگذارد.
- هر تب جداگانه ذخیره می‌شود. پیش از خروج از تب، منتظر پیام موفقیت بمانید.
- قبل از Import، تغییر تمپلیت‌ها یا اجرای عملیات Maintenance یک بکاپ به‌روز بگیرید.
- گزینه‌های مخصوص نصب باینری در نصب Docker در دسترس نیستند.

## تب Panel {#panel-tab}

<p class="rb-panel-actions"><a class="rb-panel-button" href="#" data-panel-route="/settings#panel">باز کردن تب Panel</a></p>

### فرمت پیش‌فرض لینک سابسکریپشن

این گزینه فقط تعیین می‌کند کدام قالب لینک به‌صورت پیش‌فرض در داشبورد نمایش داده شود؛ قالب‌های دیگر همچنان معتبر می‌مانند.

| فرمت | کاربرد |
| --- | --- |
| `username/key` | مسیر خوانا که نام کاربر و کلید را در خود دارد. |
| `key only` | مسیر کوتاه‌تر که نام کاربر را نمایش نمی‌دهد. |
| `token` | لینک مبتنی بر توکن برای integrationها یا flowهای فعلی کلاینت. |

### Runtime settings

| گزینه | چه چیزی را تغییر می‌دهد | ملاحظه |
| --- | --- | --- |
| **Dashboard path** | مسیر داشبورد در نصب باینری را تغییر می‌دهد؛ برای مثال `/dashboard/`. | هم‌زمان reverse proxy و bookmarkها را اصلاح کنید، وگرنه پنل ظاهراً از دسترس خارج می‌شود. |
| **Subscription read-only mode** | سابسکریپشن را بدون به‌روزرسانی metadata مربوط به آخرین استفاده ارائه می‌کند. | برای cache، monitor یا probe بیرونی مناسب است؛ اطلاعات آخرین استفاده دیگر همه fetchها را نشان نمی‌دهد. |
| **Record node usage** | تاریخچه ترافیک نود را برای صفحه Usage در دیتابیس ذخیره می‌کند. | گزارش تاریخی می‌دهد و در مقابل write و فضای دیتابیس را بیشتر می‌کند. |
| **Record user usage samples** | نمونه‌های مصرف کاربر، ادمین و سرویس را ذخیره می‌کند. | تاریخچه دقیق‌تری می‌دهد و writeهای دوره‌ای بیشتری ایجاد می‌کند. |
| **Enable API docs** | رابط OpenAPI/Swagger داخلی را روی `/docs` فعال می‌کند. | زمانی فعال کنید که ادمین یا integration نیاز به بررسی API دارد. احراز هویت و مجوزهای API همچنان اعمال می‌شوند. |

{{< callout type="info" >}}
اگر نمودار و گزارش تاریخی لازم دارید، هر دو گزینه ثبت مصرف را فعال نگه دارید. زمانی یکی را خاموش کنید که کاهش write دیتابیس از نگهداری آن تاریخچه مهم‌تر است.
{{< /callout >}}

### phpMyAdmin

این بخش برای نصب‌های MySQL و MariaDB است. می‌توانید phpMyAdmin را نصب یا فعال کنید، داخل پنل باز کنید، مسیرش را تعیین کنید و از حساب دیتابیس Rebecca یا اعتبارنامه سفارشی استفاده کنید. مسیر را خصوصی نگه دارید و اطلاعات دیتابیس را در اختیار ادمین‌های غیرضروری نگذارید.

### Maintenance

این بخش نسخه فعلی پنل و کانال آپدیت را نشان می‌دهد. در نصب باینری می‌توانید از داخل پنل Update، Soft Reload یا Restart اجرا کنید. در نصب Docker این عملیات غیرفعال‌اند، چون کنترل چرخه اجرای host باید بیرون از داشبورد انجام شود.

- **Latest** نسخه‌های پایدار را دنبال می‌کند.
- **Dev build** ممکن است تغییر یا migration ناتمام داشته باشد؛ فقط با پذیرش این ریسک از آن استفاده کنید.
- **Soft Reload** تنظیمات پنل را بدون قطع عمدی اتصال‌ها دوباره بارگذاری می‌کند.
- **Restart** برای چند ثانیه داشبورد و API را از دسترس خارج می‌کند.

## تب Backup {#backup-tab}

<p class="rb-panel-actions"><a class="rb-panel-button" href="#" data-panel-route="/settings#backup">باز کردن تب Backup</a></p>

Rebecca یک فایل قابل‌حمل با پسوند `.rbbackup` می‌سازد که میان نصب‌های SQLite، MySQL و MariaDB قابل بازیابی است.

| محدوده | داده‌های داخل بکاپ |
| --- | --- |
| **Database only** | رکوردهای دیتابیس Rebecca؛ هنگام Restore فایل‌های سرور دست‌نخورده می‌مانند. |
| **Database + Rebecca files** | دیتابیس به‌همراه فایل‌های تنظیمات و داده Rebecca، از جمله `/etc/rebecca` و `/var/lib/rebecca`. |

Import داده فعلی محدوده انتخاب‌شده را جایگزین می‌کند. پیش از تأیید، فایل و Scope را دوباره بررسی کنید و یک بکاپ سالم جداگانه نگه دارید. Export و Import عملیات سطح host هستند و فقط در حالت باینری فعال‌اند.

## تب Telegram {#telegram-tab}

<p class="rb-panel-actions"><a class="rb-panel-button" href="#" data-panel-route="/settings#telegram">باز کردن تب Telegram</a></p>

1. یک بات بسازید و **Bot API Token** آن را وارد کنید. توکن را مانند رمز عبور نگهداری کنید.
2. شناسه‌های عددی را در **Admin Chat IDs** وارد کنید. اگر لاگ‌ها باید به مقصد دیگری بروند، **Logs Chat ID** را جدا تنظیم کنید.
3. اگر مقصد Forum است، گزینه Forum را فعال و Topic IDها را مشخص کنید.
4. Proxy را فقط زمانی تنظیم کنید که سرور پنل مستقیماً به Telegram دسترسی ندارد.
5. پیش از فعال‌کردن اعلان‌ها یا بکاپ دوره‌ای، یک Test Message بفرستید.

### بکاپ دوره‌ای

ارسال زمان‌بندی‌شده بکاپ فقط در نصب باینری فعال است. Scope دیتابیس یا Full، فاصله زمانی و در صورت نیاز چت اختصاصی بکاپ را تعیین کنید. **Send backup now** سریع‌ترین راه برای بررسی مجوز بات و دریافت فایل است.

### اعلان‌ها

گروه‌های اعلان رویدادهای کاربر، ادمین، نود، ورود و خطا را پوشش می‌دهند. فقط رویدادهایی را فعال کنید که نیاز به اقدام دارند؛ ارسال همه رخدادها در یک چت، هشدارهای مهم را میان پیام‌های عادی پنهان می‌کند.

## تب Subscriptions {#subscriptions-tab}

<p class="rb-panel-actions"><a class="rb-panel-button" href="#" data-panel-route="/settings#subscriptions">باز کردن تب Subscriptions</a></p>

تنظیمات Global سابسکریپشن برای همه ادمین‌ها اعمال می‌شود، مگر برای ادمینی که Override جداگانه دارد.

- **Subscription URL prefix** دامنه عمومی لینک‌های تولیدشده را مشخص می‌کند. برای URL نسبی خالی بگذارید.
- **Custom templates directory** مسیر بارگذاری تمپلیت‌های Jinja را تغییر می‌دهد.
- **Subscription profile title**، **Support URL** و **Profile update interval** برای کلاینت‌های سازگار ارسال و در صفحه سابسکریپشن نمایش داده می‌شوند.
- فیلدهای Template فایل خروجی صفحه سابسکریپشن، Home، Clash، V2Ray، Happ، Incy، Sing-box و Mux را تعیین می‌کنند.
- **Subscription alias URLs** مسیرهای سازگار اضافی را خط‌به‌خط تعریف می‌کند.
- **Subscription ports** پورت‌های اضافی را به URLهای سابسکریپشن تولیدشده اضافه می‌کند.
- گزینه‌های Client JSON مشخص می‌کنند Rebecca برای کلاینت‌های پشتیبانی‌شده از رفتار JSON سفارشی استفاده کند یا نه.

Override ادمین باید استثنا باشد. مقادیر مشترک را Global نگه دارید، فقط برای ادمین لازم دامنه یا تمپلیت متفاوت تعریف کنید و برای بازگشت به مقدارهای اصلی از **Reset overrides** استفاده کنید.

## تب Template Creator {#template-creator-tab}

<p class="rb-panel-actions"><a class="rb-panel-button" href="#" data-panel-route="/settings#template-creator">باز کردن Template Creator</a></p>

Template Creator صفحه سابسکریپشن را با ویجت‌های Drag & Drop می‌سازد. پیش از ذخیره، نتیجه را در عرض دسکتاپ و موبایل Preview کنید. خود پنل این تب را فعلاً در حال توسعه فعال معرفی می‌کند؛ بنابراین یک کپی از تمپلیت سالم نگه دارید و Draft آزمایش‌نشده را پیش‌فرض محیط عملیاتی نکنید.

## ترتیب پیشنهادی برای پنل تازه

1. مسیر داشبورد و رفتار پیش‌فرض سابسکریپشن را در **Panel** تنظیم کنید.
2. از تب **Backup** یک بکاپ پایه بگیرید.
3. Telegram را تست کنید و بعد اعلان رویدادها را فعال کنید.
4. مقادیر Global سابسکریپشن را مشخص کنید و فقط Overrideهای ضروری را بسازید.
5. خروجی Template Creator را تا زمان بررسی با لینک واقعی کلاینت، Draft در نظر بگیرید.
