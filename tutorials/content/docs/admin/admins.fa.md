## معرفی {#section-admins-page-intro}

جدول نقش، وضعیت، سقف دیتا/کاربر و آمار سریع را نشان می‌دهد؛ با فیلتر سریع پیدا کنید.

## آموزش مدیریت {#section-admins-page-guide}

1. دکمه «افزودن ادمین» را بزنید، نام کاربری/رمز را وارد کنید و یکی از رول‌های Full Access / Sudo / Reseller / Standard را انتخاب کنید.
2. سقف دیتا (data_limit) و سقف کاربران (users_limit) را تعیین کنید تا محدوده‌ی فعالیت او مشخص باشد.
3. با enable/disable می‌توانید دسترسی ادمین و کاربران زیرمجموعه‌اش را موقتاً متوقف یا فعال کنید.
4. حذف، ادمین و کاربرانش را به صورت نرم (soft delete) غیر‌فعال می‌کند؛ برای ایجاد سریع از دکمه میانبر استفاده کنید.

{{< callout type="info" >}}
**نکته‌ها**

- بعد از بالا بردن سقف‌ها یا reset usage، اگر هنوز متوقف است، دوباره enable کنید.
- برای تیم پشتیبانی از Sudo/Standard استفاده کنید و Full Access را محدود نگه دارید.
{{< /callout >}}

## فول اکسس {#section-admins-page-role-full-access}

مالک اصلی با دسترسی کامل به همه بخش‌ها و تنظیمات حساس (Users, Admins, Services, Hosts, Nodes, Integrations و تنظیمات).

1. تعریف/ویرایش رول‌ها، سرویس‌ها، نودها و ادغام‌ها بدون محدودیت.
2. تعیین سقف و دسترسی برای سایر ادمین‌ها و کاربران.
3. برای مالک یا مدیر اصلی نگه دارید؛ تعداد کم.

{{< callout type="info" >}}
**نکته‌ها**

- قبل از واگذاری این رول، دسترسی‌ها را دوباره بررسی کنید.
- برای تغییرات حساس، از 2FA و رمز قوی استفاده کنید.
{{< /callout >}}

## سودو {#section-admins-page-role-sudo}

دسترسی عملیاتی بالا بدون مالکیت کامل؛ برای تیم فنی/پشتیبانی ارشد.

1. ساخت/ویرایش کاربران، سرویس‌ها و نودها (بجز تنظیمات مالکانه).
2. می‌تواند بیشتر عملیات روزمره را انجام دهد اما تغییرات مالکیتی محدود است.
3. تعداد را محدود نگه دارید تا ریسک کمتر شود.

{{< callout type="info" >}}
**نکته‌ها**

- اگر نیاز به مدیریت Full Access ندارند، این رول کافی است.
- سقف‌ها و گزارش‌ها را دوره‌ای بررسی کنید.
{{< /callout >}}

## ریسلر {#section-admins-page-role-reseller}

فروشنده با سقف دیتا/کاربر مشخص؛ فقط کاربران خودش را مدیریت می‌کند.

1. ایجاد و مدیریت کاربران زیرمجموعه خودش در محدوده users_limit و data_limit.
2. دسترسی به بخش‌های حساس سیستم ندارد؛ فقط آنچه برای فروش نیاز است.
3. برای نماینده‌ها یا همکاران محدود استفاده کنید.

{{< callout type="info" >}}
**نکته‌ها**

- کاربران on hold در users_limit حساب نمی‌شوند؛ فعال‌ها حساب می‌شوند.
- سقف‌ها را متناسب با قرارداد تنظیم کنید.
{{< /callout >}}

## استاندارد {#section-admins-page-role-standard}

کمترین سطح؛ فقط بخش‌هایی که برایش روشن کرده‌اید را می‌بیند/ویرایش می‌کند.

1. مناسب کارآموز یا پشتیبان محدود.
2. دسترسی هر بخش (Users, Admins, Services, ...) باید جداگانه فعال شود.
3. Self permissions مشخص می‌کند کارت‌های «حساب من» نمایش داده شود یا نه.

{{< callout type="info" >}}
**نکته‌ها**

- اگر فقط مشاهده می‌خواهید، can_view را فعال و can_edit را غیرفعال بگذارید.
- برای ارتقا، دسترسی‌ها را مرحله‌ای اضافه کنید.
{{< /callout >}}
